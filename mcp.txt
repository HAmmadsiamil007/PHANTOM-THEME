## 2. MCP Servers

| Server | Protocol | Purpose |
|--------|----------|---------|
| **Headroom** | MCP | Conversation compression — compress history, memory, large tool outputs |
| **Context7** | MCP | Library/framework documentation lookup. Always use before implementing unfamiliar APIs |
| **Stitch MCP** | MCP | Google Stitch design platform — init, proxy, screens, upload, site build, snapshot, doctor |
| **Auto Browser MCP** | MCP | Playwright browser control with human takeover — for browser testing, QA, form fills |
| **Autobrowse** | MCP | Self-improving browser automation with auto-research loop |
| **Serena** | MCP | Semantic code retrieval, editing, refactoring. AI-native IDE toolkit with memories, modes, LSP. `serena init`, `serena start-mcp-server` |

### 2.1 MCP Configuration Locations

- Headroom + Context7: configured via `.opencode\` or global OpenCode MCP config
- Stitch MCP: `stitch-mcp init` | `stitch-mcp proxy` | `stitch-mcp doctor`
- Auto Browser MCP: `C:\Users\hammad\Downloads\uiux\docker compose up` or `uvx auto-browser-mcp`
- Autobrowse: `~/.agents/skills/autobrowse/SKILL.md`
- Serena: `serena init` | `serena start-mcp-server --project-from-cwd` | Config: `~/.serena/serena_config.yml` | `serena help`

## 7. Quick Reference

### 7.1 Skill Loading Order

1. Load 6 core systems (karpathy, headroom, caveman, context7, impeccable, loop-engineering)
2. Read mcp.txt (this file)
3. Load task-specific skills from use case matrix above
4. Load Context7 docs for any unfamiliar API/framework
5. If no installed skill fits → find-skills

### 7.2 Naming Conventions

- Project skills: `.agents/skills/<name>/`
- Global skills: `~/.agents/skills/<name>/`
- Anthropic sub-skills: `.agents/skills/anthropic-skills/skills/<name>/`
- Stitch skills: `~/.agents/skills/stitch-<name>/`
- Shop: always type `Shopify` (capital S)
- For MCP servers prefixed with `mcp__`, use `@modelcontextprotocol/server-*` style
